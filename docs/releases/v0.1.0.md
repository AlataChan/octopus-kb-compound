# octopus-kb-compound v0.1.0

`octopus-kb-compound` packages a repeatable pattern for self-growing LLM knowledge bases:

- `kb-retrieve` for evidence-backed retrieval against a persistent wiki
- `kb-maintain` for ingest, metadata, links, and graph hygiene
- `obsidian-graph` prompts for stable wikilinks and canonical pages
- a zero-dependency Python toolkit for frontmatter, links, linting, vault scanning, and profile-aware CLI operations

## Release Highlights

- GitHub-ready repository layout with docs, tests, CI, and contribution guidance
- Minimal example vault that demonstrates schema, index, log, concept, entity, and raw evidence layers
- Production-vault support through `.octopus-kb.yml` profiles
- Link resolution that handles aliases, file names, and relative-path wikilinks used in real Obsidian vaults
- Lint behavior hardened for real content by ignoring code-block pseudo links and folder placeholders

## Migration Note

This release also introduces the first production profile pattern for an existing LLM vault:

- root `AGENTS.md` as the operator entry point
- root `.octopus-kb.yml` for scan scope and graph boundaries
- `wiki/LOG.md` as the append-only maintenance trail
